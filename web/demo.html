Steps — surface vs backstage

分步 — 表面上 / 背后

1. Install

Surface — Copy web-learning-github/ into your host skills path.
Backstage — Host scans for SKILL.md; rules in references/ apply only when generating HTML.

2. Generate

Surface — You type the target owner/repo in chat.
Backstage — Model may call GitHub API for stats; builds journey + tree from repo layout.

1. 安装

表面上 — 把 web-learning-github/ 拷进宿主要求的 skills 目录。
背后 — 宿主查找 SKILL.md；生成 HTML 时才读取 references/。

2. 生成

表面上 — 在对话里给出目标 owner/repo。
背后 — 模型可能请求 GitHub API；按目录结构拼 journey 与树。

README angles (illustrative)

README 侧写（示意）

  • Demand — Skill repos are easy to star, hard to “see the wiring.”
  • Distribution — One sharable HTML; works offline after fonts cache.
  • Moat / caveats — Snapshot stats; regenerate when the repo moves.
  • Compliance — Don’t paste secrets into generated pages.
  • 需求 — Skill 仓好收藏，调用链却不直观。
  • 传播 — 单文件 HTML，字体缓存后可离线传阅。
  • 注意 — Star 等为快照；仓库大变要重新生成。
  • 合规 — 勿把密钥写进生成页。
