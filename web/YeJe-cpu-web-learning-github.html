Steps — surface vs backstage

分步 — 表面上 / 背后

1. Install the skill folder

Surface — You drop web-learning-github/ into the host’s skills directory (e.g. Cursor .agents/skills/web-learning-github/).
Backstage — The host indexes SKILL.md; references/* stay on disk until a generation task runs.

2. Trigger with owner/repo + language prefs

Surface — Natural language: map repo, bilingual toggle, default output under web/.
Backstage — Model follows SKILL ordering: meta → blurbs → journey → steps → tree → bullets; chat messages start hidden and reveal on Play.

3. Ship or archive the HTML

Surface — Double-click the file, scroll, screenshot, or attach to docs.
Backstage — No runtime server required; fonts may load once from Google Fonts then cache offline.

1. 安装 skill 目录

表面上 — 将 web-learning-github/ 放进宿主约定的 skills 路径（如 Cursor 的 .agents/skills/web-learning-github/）。
背后 — 宿主识别 SKILL.md；references/* 在真正生成 HTML 时才会被读到。

2. 用自然语言触发

README angles

README 侧写

  • Demand — Skill repos hide real entrypoints under references/ and naming conventions; one page reduces tab fatigue.
  • Distribution — Single-file HTML is easy to share or drop in web/ without a build step.
  • Caveat — GitHub counters are a snapshot at generation time.
  • Compliance — Do not embed secrets or private URLs in generated walkthroughs.
  • 需求 — Skill 仓的真实入口常在 references/ 与约定名之下，单页能减少来回切 Tab。
  • 传播 — 单文件 HTML 易分享，放进 web/ 也无须构建链。
  • 注意 — GitHub 数字为生成时刻的快照。
  • 合规 — 勿在生成页中写入密钥或内网地址。
